# 灵光收集箱改动记录

记录范围从 `apps/spark-inbox-preview/preview-b.html` 开始，到当前原生 Android 版本为止。

## 2026-05-20

### `preview-b.html`：时间线草案

- 做出第一版更轻的 HTML 预览，方向从普通备忘录收敛到“快速记录 + 回头处理”。
- 页面仍保留明显标题“灵光收集箱”，整体更像时间线备忘录。
- 核心能力包括快速输入、保存、本地记录列表，以及未处理/已完成的简单筛选。
- 这一版主要验证“打开就写”和“之后再处理”的基本流程。

### `preview-v2.html`：状态模型成型

- 记录类型被统一，不再区分“想法”和“待做”。
- 引入四种状态：未处理、进行中、已完成、已放弃。
- 首页开始承担更明确的快速捕获职责，下方展示进行中的记录。
- 查看页集中承载记录列表、状态筛选和搜索。
- 增加已放弃入口，让舍弃内容被收纳，而不是占用主列表。

### `index.html`：交互和布局收敛

- 首页删除复杂的最近捕获区域，只保留日期、输入区、保存动作和最多 3 条进行中记录。
- 保存按钮移动到屏幕中线附近，避免首页需要向下滑。
- 进行中记录改为竖向排列。
- 圆环成为主要状态操作：未处理 -> 进行中 -> 已完成；已完成/已放弃可重新打开。
- 移除和圆环重复的按钮，只保留编辑、放弃、放回未处理、删除等辅助操作。
- 支持记录内容编辑。
- 查看页支持全部、未处理、进行中、已完成、已放弃筛选，并支持搜索记录内容。
- 方向明确为轻量手机入口：手机负责快速记录和提醒自己，深度整理仍放到电脑和 Obsidian。

### `spark-inbox-native`：原生 Android MVP

- 新建原生 Android 项目 `apps/spark-inbox-native`。
- 使用 Java 原生 View 实现，不依赖 Gradle，沿用参考项目的 Android SDK 直构建方式。
- 数据使用 `SharedPreferences` 本地保存。
- 实现两页结构：`记` 用于快速记录，`看` 用于查看、搜索和处理。
- 实现记录字段：内容、状态、创建时间、开始时间、完成时间、放弃时间、更新时间。
- 实现首页最多展示 3 条进行中记录。
- 实现记录编辑、删除、放弃、放回未处理和圆环快捷状态流转。
- 添加 `build-apk.ps1`，输出 `dist/spark-inbox-debug.apk`。
- 真机安装并完成冒烟测试：启动、保存、查看、搜索、状态流转、放弃、编辑、删除均通过。

### 当前 UI 调整

- 首页时间字号放大，并整体下移。
- 首页输入区改为带边框的深色输入框，视觉边界更明确。
- 删除 `0 / 500` 字数计数。
- 保存按钮保留在输入区下方并靠右对齐。
- 第二页删除顶部”记录”标题，让搜索框成为页面入口。
- 重新构建并覆盖安装到手机，真机确认首页和第二页布局已生效。

### 灵感分类 + 三 tab 重构

- `InboxRecord` 新增 `type` 字段（task/idea），与现有状态机共存。
- `RecordStore` 迁移到 `records_v2` key，种子数据包含 5 条任务和 2 条灵感。
- 底部导航从两 tab（记/看）改为三 tab（记/任务/灵感）。
- 首页新增类型选择器（任务/灵感），保存时按选定类型创建记录。
- 任务视图保留四状态筛选（全部/未处理/进行中/已完成/已放弃）。
- 灵感视图新增两种状态：未处理/已整理，用紫色圆环标识。
- `build-apk.ps1` 修复编译目录权限锁定问题，改用 `$env:TEMP` 构建。
- 首页布局调整：顶部内边距 dp(48)→dp(40)，日期底部间距缩小，输入区 minLines 7→5。
- 输入卡和任务区权重重新分配（0.85:1.05），三个进行中任务不再被压缩。
- 底部导航高度缩减并添加底部内边距，避开系统导航栏重叠。
- 真机安装并验证：三 tab 切换、任务/灵感分别创建和筛选、首页三个任务完整显示。

### 打包目录整理

- 保留 `apps/spark-inbox-preview` 作为唯一主项目，原生 Android 源码以 `native/app` 为准。
- 删除旧独立副本 `apps/spark-inbox-native` 和旧临时构建目录 `apps/spark-inbox-native-build`。
- 删除历史遗留的根目录 debug keystore。
- `native/build-apk.ps1` 改为使用系统临时目录编译，并在构建成功后自动清理临时目录。
- debug keystore 移到 `apps/.android-debug/`，不再生成到项目目录。
- 项目内只保留 `native/dist/` 作为 APK 输出目录，且已被 Git 忽略。
